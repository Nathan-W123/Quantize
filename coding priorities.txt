Coding Priorities
=================

1) Build reusable multi-start orchestration (highest priority)
- OCS implementation should be treated as a prototype.
- Move multi-start logic into a shared module: `multistart.py`.
- Implement a reusable `run_multistart(...)` function used by all molecules.
- Refactor `run_water.py`, `run_OCS.py`, and `run_n.py` to call `run_multistart(...)`.
- Standardize best-run selection and consensus uncertainty output (mean/std).

2) Improve efficiency / runtime performance
- Reduce Hessian frequency with smarter scheduling (adaptive recalc strategy).
- Reuse curvature info safely between iterations when geometry drift is small.
- Add stronger early-stop criteria for spectral/energy stall conditions.
- Add caching for repeated derivative evaluations at near-identical geometries.
- Parallelize multi-start runs across CPU cores.
- Add optional fast pre-optimization stage before expensive quantum updates.

3) General-case robustness for any molecule
- Keep and extend auto-sanitization of unstable spectral components.
- Improve geometry initialization from `guess_geometry(elems, bonds)` for rings/branching.
- Add automatic fallback weighting/masking when components are ill-conditioned.
- Add stronger input validation for isotopologue schema, units, and dimensions.

4) Production-grade output and diagnostics
- Emit structured machine-readable results (JSON) per run.
- Include convergence status, rank, residuals, energy trajectory, and uncertainty.
- Record per-start outcomes for auditability and troubleshooting.
- Add clear error categories (data inconsistent, stalled, backend failure, etc.).

5) Accuracy upgrades (without requiring extra isotopologues)
- Improve default quantum prior level (method/basis presets by molecule size).
- Add optional two-tier workflow: fast method pre-pass + higher-level refinement.
- Support uncertainty reporting as first-class output (not only point geometry).

6) Validation and CI
- Build a molecule benchmark suite (small, medium, edge cases).
- Add regression tests for convergence behavior and output schema stability.
- Track wall-time metrics in CI to detect efficiency regressions.
